# Спека: экспорт грида смен в PDF / PNG / CSV (Мини-этап 1.5)

- **Дата:** 2026-06-12
- **Статус:** утверждена основателем (брейнсторм-сессия)
- **Связано:** ROADMAP.md → Этап 1.5; обещание лендинга: «Экспорт PDF и PNG — один клик, готово к печати, отправке в чат или вложению в письмо» (`marketing.whyus.i6`); тарифы: экспорт доступен на всех планах, включая Старт (`pricing/page.tsx` → `feature_export: true` у всех)

## Цель

Кнопка «Экспорт» в тулбаре грида (как в демо на лендинге, где она сейчас фейковый тост) выгружает месяц графика в PDF, PNG или CSV. Без серверной части: данные месяца уже загружены в TanStack Query на клиенте.

## Ключевые решения (утверждены)

1. **Подход A: клиентский рендер.** Скрытый экспорт-рендерер (таблица месяца без виртуализации и интерактива, те же дизайн-токены) → `html-to-image` для снимков, `jspdf` для сборки PDF. Обе библиотеки подгружаются динамическим `import()` в момент экспорта — главный бандл не растёт. Данные не покидают браузер; серверных затрат и новых запросов к БД нет.
2. **Диалог с выбором перед экспортом** (решение основателя, осознанное отступление от «одного клика»): охват + формат, кнопка «Скачать».
3. **CSV входит в скоуп** — лендинг обещает его тарифу «Старт» (`marketing.plans.startF4`). Генерация из загруженной `Map` без библиотек.
4. **План-гейтинг не нужен:** матрица прайсинга даёт экспорт всем планам. Read-only-триал и роль viewer тоже могут экспортировать — это чтение.

## UX-поток

- Кнопка «Экспорт» в тулбаре грида, рядом с существующими контролами; видна всем ролям; в полностью пустой организации тулбар и так скрыт.
- Модалка (Radix-диалог, как существующие модалки грида):
  - **Охват:** «Весь месяц» (дефолт) / «Текущий вид» — показывается только при активном фильтре отдела или поиске / «Конкретный отдел» — select по отделам.
  - **Формат:** PDF / PNG / CSV.
- «Скачать» → прогресс на кнопке (блокируется только она) → скачивание файла → тост «График выгружен» → модалка закрывается.
- Имя файла латиницей без локализации: `smengo-schedule-2026-06.pdf`, при выборе отдела — `smengo-schedule-2026-06-<dept-slug>.pdf` (slug — транслитерация/санитизация названия отдела).
- Режим ячеек в PDF/PNG = текущий режим тулбара (compact / detail / extended) — WYSIWYG.

## Архитектура

| Юнит | Ответственность |
|---|---|
| `components/schedule/export/export-modal.tsx` | Модалка: выбор охвата/формата, оркестрация генерации, прогресс, тосты. Данные месяца и отделы — через props от грида |
| `components/schedule/export/export-renderer.tsx` | «Чистая» таблица месяца: шапка (организация, месяц, дата генерации), строки без виртуализации, легенда статусов. Умеет рендерить подмножество строк (для постраничного PDF). Монтируется в портал за экраном только на время генерации |
| `lib/export/csv.ts` | `buildCsv()` — чистая функция, юнит-тестируемая |
| `lib/export/paginate.ts` | `paginateRows()` — нарезка строк по страницам A4, чистая функция |
| `lib/export/filename.ts` | сборка имени файла + slug отдела |
| `lib/export/download.ts` | `downloadBlob()` — скачивание через временный `<a>` |

Новых таблиц, миграций и server actions нет. PostHog-событие `schedule_export` `{format, scope, rows}`.

## Форматы

### PDF
- A4 landscape. Каждая страница — отдельный снимок рендерера с подмножеством строк: шапка дней повторяется на каждой странице, строки не режутся, на первой странице заголовок (организация + месяц), внизу номер страницы.
- `paginateRows()` считает строки на страницу из фиксированной высоты строки.
- Постраничные снимки заодно обходят лимит площади canvas в Safari.

### PNG
- Один цельный снимок всей таблицы (удобнее для чата/печати одним листом).
- `pixelRatio` динамический: 2× по умолчанию, автоматически снижается так, чтобы `width × height × ratio² ≤ ~16,7 млн px²` (лимит Safari). На 300 сотрудниках получится ~1× — читаемо.

### CSV
- Строки — сотрудники (ФИО, отдел, должность); колонки — дни месяца; ячейка — код статуса + время смены при наличии: `W 09:00–18:00`.
- **UTF-8 с BOM** и разделитель `;` — иначе Excel в кириллических локалях ломает кодировку и колонки.
- **Защита от formula injection:** значения, начинающиеся с `=`, `+`, `-`, `@`, префиксуются `'` — имена сотрудников и кастомных статусов вводит пользователь.

## Ошибки и границы

- Сбой генерации → существующий тост ошибки + `Sentry.captureException`.
- Пустой месяц/отдел — честный экспорт с пустыми ячейками, не ошибка.
- Во время генерации грид остаётся живым; блокируется только кнопка «Скачать».

## i18n

Все строки модалки, шапки экспорта и легенды — через next-intl, ключи в `messages/{ru,uk,en}.json` одновременно. Тост «График выгружен» уже есть в demo-неймспейсе — в app-неймспейс добавляется свой ключ.

## Тестирование (Definition of Done)

1. Юниты: `buildCsv` (BOM, `;`, кириллица, времена, экранирование формул), `paginateRows` (граничные случаи), `filename` (slug кириллических отделов).
2. Компонентный смоук: export-renderer рендерит N строк без виртуализации (happy-dom).
3. tsc + vitest + production build чистые.
4. Живой E2E в превью на «Claude E2E Test Org»: скачать все три формата; PDF/PNG проверить визуально (скриншот), CSV открыть и проверить кодировку/разделитель.
5. Перф: экспорт PDF и PNG на 300×30 не подвешивает вкладку (прогресс виден, результат скачивается).
6. **🔒 `/security-review` после этапа:** CSV-инъекции, утечка данных чужих организаций в экспорт, санитизация имени файла.

## Вне скоупа

- Серверная генерация и шаринг по ссылке.
- Экспорт диапазонов ≠ месяц, нескольких месяцев, выбранных сотрудников.
- Брендирование экспорта логотипом организации (нет поля логотипа в БД).
- Правки маркетингового копи не нужны: после реализации и «Экспорт в CSV» у Старта (главная), и «Экспорт PDF / PNG» (прайсинг) становятся правдой.
